git

Install or Update

Install

sudo apt-get install git git-core

Update

git clone git://git.kernel.org/pug/scm/git/git.git

Create a local repository

  1. mkdir my_app
  2. cd my_app
  3. git init // init current repository
  4. git add . // add sources to repository in current dir
  5. git commit -m "Add the original files" // commit files to repo
  6. git show // show status
  7. git log // show all changes we've committed so far

Remote Repositories

  1. Create a new empty GitHub repository
  2. Put our local repo to the GitHub server, we'll need to add a remote repository
    git remote add origin https://github.com/Ryan311/MyWiki.git
  3. push our local changes to origin repo on Github
    git push -u origin master //-u tells Git to remember the para
  4. Pulling Remotely other people commit and we want to see the changes
    git pull origin master
  5. git push // push everything to the remote repository

Differences

  1. git diff HEAD
  2. git diff --staged

Other

  1. git reset octofamily/octodog.txt
  2. git checkout -- octocat.txt
  3. git brahch clean_up
  4. git checkout clean_up
  5. git rm '*.txt' //removing all the things

ssh

  1. ssh-keygen
  2. copy ~/.ssh/id_rsa.pub to github
  3. ssh -T git@github.com

work with two computers

  1. Computer A (had the original git repo)
  2. Computer B (has the cloned repo)
  3. git push origin master (make some changes on computer A)
  4. git pull origin master (go to computer B)
  5. git push origin master

ssh连接github.com的SSH服务

  • $ssh-keygen ==> 在.ssh目录中创建默认的公钥／私钥对文件
  • $ssh-keygen -C "persnail311@gmail.com" -f ~/.ssh/hr ==> 在目录hr下根据邮箱名创建公钥／私钥对文件，还要在.ssh/config文件中加入路径名
  • 将公钥Copy到GitHub中
  • $ssh -T git@github.com ==> 登录
  • $ssh -Tv git@github.com ==> 以诊断模式登录
